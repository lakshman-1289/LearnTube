## **Key Concepts**
- **Java Development Kit (JDK)**: Software environment for building Java applications (includes compiler, Java Runtime Environment, and reusable code).
- **Code Editor/IDE**: Tools like IntelliJ IDEA, Eclipse, or NetBeans for writing and managing Java code.
- **Functions/Methods**:
  - A block of code that performs a specific task.
  - In Java, functions inside classes are called *methods*.
- **Classes**: Containers for related methods (functions) used to organize code.
- **Main Method**: The entry point of a Java program (`public static void main(String[] args)`).
- **Access Modifiers**: Keywords like `public` or `private` that control visibility of classes/methods.
- **Naming Conventions**:
  - **PascalCase** for classes (e.g., `HelloWorld`).
  - **camelCase** for methods (e.g., `sendEmail`).
- **Packages**: Used to group related classes (e.g., `com.codewithmosh`).

---

## **Important Points**

### **1. Setting Up Java**
- **JDK Installation**:
  - Download from [Oracle’s Java SE page](https://www.oracle.com/java/technologies/javase-downloads.html).
  - Accept the license agreement and install for your OS (Windows/macOS/Linux).
- **IDE Setup**:
  - IntelliJ IDEA (Community Edition is free) is recommended for beginners.
  - Drag the downloaded `.dmg` (macOS) or `.exe` (Windows) to the Applications folder.

### **2. Anatomy of a Java Program**
- **Structure**:
  ```java
  public class Main {                  // Class (PascalCase)
      public static void main(String[] args) {  // Main method (entry point)
          System.out.println("Hello World!");   // Code to execute
      }
  }
  ```
- **Key Components**:
  - **Class**: Must match the filename (e.g., `Main.java`).
  - **Main Method**: Required to run the program.
  - **Curly Braces `{}`**: Define the scope of classes/methods.
  - **Semicolons `;`**: Terminate statements.

### **3. Functions vs. Methods**
- **Function**: A standalone block of code (e.g., in Python).
- **Method**: A function inside a class (Java’s version of functions).
  - Example:
    ```java
    public void sendEmail(String recipient, String subject) {
        // Code to send email
    }
    ```
  - **Return Type**:
    - `void`: No return value.
    - `int`, `String`, etc.: Returns a value of that type.

### **4. Access Modifiers**
- **`public`**: Accessible from any other class.
- **`private`**: Accessible only within the same class.
- Most classes/methods in beginner programs are `public`.

### **5. Packages**
- Organize classes into namespaces (e.g., `com.codewithmosh`).
- Convention: Reverse domain name (e.g., `com.google` for Google).

### **6. Writing Your First Program**
1. Open IntelliJ IDEA → Create a new Java project.
2. Select "Command Line App" template.
3. Name the project (e.g., `HelloWorld`).
4. Set the base package (e.g., `com.yourname`).
5. Write the code in `Main.java`:
   ```java
   public class Main {
       public static void main(String[] args) {
           System.out.println("Hello World!");
       }
   }
   ```
6. Run the program (▶️ button in IntelliJ).

## **Key Concepts**

---

## **Important Points**

### **1. Java Project Structure**
- A Java project contains folders and files organized in a hierarchical structure.
- **Project Panel**: Displays all folders and files in the project (e.g., `HelloWorld` project).
  - **Source Folder (`src`)**: Contains the source code of the project.
    - **Package**: A namespace that organizes related classes (e.g., `com.codewithmosh`).
      - **Class File**: Contains Java code (e.g., `Main.java`).

### **2. Java File Basics**
- All Java files must have the `.java` extension (e.g., `Main.java`).
- A Java file typically starts with a **package statement**, which specifies the package the class belongs to.
  - Example: `package com.codewithmosh;`
  - The package statement **must end with a semicolon (`;`)**.

### **3. Class and Method Structure**
- A Java program consists of **classes** and **methods**.
  - **Class**: Defined using the `class` keyword (e.g., `public class Main { }`).
    - The class name should match the file name (e.g., `Main.java` contains `public class Main`).
  - **Method**: A block of code that performs a specific task.
    - The `main` method is the **entry point** of a Java program:
      ```java
      public static void main(String[] args) { }
      ```
      - `public`: Accessible from other parts of the program.
      - `static`: Belongs to the class (not an instance of the class).
      - `void`: Does not return a value.
      - `String[] args`: Parameters passed to the program (used for command-line arguments).

### **4. Writing and Running Code**
- **Comments**: Used to explain code and are ignored by the compiler.
  - Single-line comment: `// This is a comment`
- **Printing Output**: Use `System.out.println()` to print text to the terminal.
  - Example: `System.out.println("Hello World");`
    - `System`: A class in the `java.lang` package.
    - `out`: A field of the `System` class (type `PrintStream`).
    - `println()`: A method of the `PrintStream` class that prints a line of text.
  - **String**: A sequence of characters enclosed in double quotes (`" "`).

### **5. Compilation and Execution**
- Java programs undergo **two main steps** to run:
  1. **Compilation**:
     - The Java compiler (`javac`) converts `.java` source code into **Java bytecode** (`.class` file).
     - Bytecode is **platform-independent** (can run on any OS with a JVM).
     - Example command: `javac Main.java`
  2. **Execution**:
     - The **Java Virtual Machine (JVM)** executes the bytecode.
     - The JVM translates bytecode into **native code** (machine code specific to the OS).
     - Example command: `java com.codewithmosh.Main`
       - The full path to the class must include the package name (e.g., `com.codewithmosh.Main`).

### **6. Java Platform Independence**
- Java is **platform-independent** because of its **"Write Once, Run Anywhere" (WORA)** architecture.
  - **Bytecode**: Generated during compilation and can run on any device with a JVM.
  - **JVM**: Acts as an interpreter, converting bytecode to native code for the underlying OS.
- Other languages like **C#** and **Python** also use a similar architecture.

### **7. Java Editions**
Java has **four editions** for different use cases:
1. **Java Standard Edition (Java SE)**:
   - Core Java platform used for general-purpose programming.
   - Contains essential libraries for all Java developers.
2. **Java Enterprise Edition (Java EE)**:
   - Used for building large-scale, distributed systems.
   - Built on top of Java SE with additional libraries.
3. **Java Micro Edition (Java ME)**:
   - Subset of Java SE designed for mobile and embedded devices.
4. **Java Card**:
   - Used for programming smart cards.

### **8. Java History and Facts**
- Developed by **James Gosling** in **1995** at **Sun Microsystems** (acquired by Oracle in 2010).
- Originally named **"Oak"** (after an oak tree), then **"Green"**, and finally **"Java"** (inspired by Java coffee).
- Latest version (as of the transcript): **Java SE 12** (released March 2019).
- **Statistics**:
  - ~9 million Java developers worldwide.
  - Runs on **3 billion mobile phones**, **120 million TV sets**, and **every Blu-ray player**.
  - Average salary of a Java developer in the US: **$100,000+ per year**.

### **Key Concepts**

- **Variables**: Used to temporarily store data in memory.
- **Data Types in Java**:
  - **Primitive Types**: Store simple values (e.g., `int`, `double`, `boolean`, `char`).
  - **Reference Types**: Store complex objects (e.g., strings, arrays, custom objects).
- **Variable Declaration & Initialization**:
  - Syntax: `type name = value;`
  - Must be initialized before use.
- **Naming Conventions**: Use **camelCase** for variable names (e.g., `myAge`, `viewsCount`).
- **Type Conversion (Casting)**: Converting one data type to another.
- **Clean Code Principles**: Use meaningful variable names and avoid multiple declarations on one line.
- **Project**: Build a **mortgage calculator** using learned concepts.

---

### **Important Points**

#### **1. Variables in Java**
- Variables are **containers** that hold data in memory.
- Syntax:
  ```java
  int age = 30; // type identifier = value;
  ```
- **Initialization**: Must assign a value before using a variable.
- **Reassignment**: Can change the value of a variable after declaration.
  ```java
  age = 35; // Updates the value
  ```
- **Copying Values**:
  ```java
  int myAge = 30;
  int herAge = myAge; // herAge now holds 30
  ```

#### **2. Naming Conventions (Best Practices)**
- Use **camelCase** for variable names (e.g., `studentName`, `isEligible`).
- Avoid single-letter names (e.g., `x`, `n`)—use descriptive names (e.g., `viewsCount`).
- Cannot use **Java reserved keywords** (e.g., `int`, `class`, `public`) as variable names.

#### **3. Primitive Data Types**
Used for storing simple values. Java has **8 primitive types**:

| Type    | Size (Bytes) | Range / Usage                          | Example                     |
|---------|--------------|----------------------------------------|-----------------------------|
| `byte`  | 1            | -128 to 127                            | `byte age = 30;`            |
| `short` | 2            | -32,768 to 32,767                      | `short temperature = 20;`   |
| `int`   | 4            | -2B to 2B                              | `int viewsCount = 1_000_000;` |
| `long`  | 8            | Very large numbers                     | `long population = 8_000_000_000L;` |
| `float` | 4            | Decimal numbers (less precise)         | `float price = 10.99F;`     |
| `double`| 8            | Decimal numbers (more precise)         | `double salary = 50000.50;` |
| `char`  | 2            | Single Unicode character               | `char grade = 'A';`         |
| `boolean`| 1 bit       | `true` or `false`                      | `boolean isEligible = true;`|

- **Notes**:
  - Use underscores (`_`) for readability in large numbers (e.g., `1_000_000`).
  - Append `L` for `long`, `F` for `float` (e.g., `100L`, `3.14F`).
  - `char` uses **single quotes** (`'A'`), `String` uses **double quotes** (`"Hello"`).

#### **4. Reference Types**
- Used for **complex objects** (e.g., strings, arrays, custom classes).
- Example:
  ```java
  String name = "Alice"; // String is a reference type
  ```
- More details in later sections.

#### **5. Clean Code Practices**
- Declare **one variable per line** for readability.
- Use **meaningful names** (e.g., `viewsCount` instead of `v`).
- Avoid abbreviations unless widely understood.

---

### **Examples**

#### **Example 1: Declaring and Using Variables**
```java
public class Main {
    public static void main(String[] args) {
        int age = 30;          // Declare and initialize
        System.out.println(age); // Output: 30

        age = 35;              // Reassign
        System.out.println(age); // Output: 35

        int myAge = 30;
        int herAge = myAge;    // Copy value
        System.out.println(herAge); // Output: 30
    }
}
```

#### **Example 2: Primitive Types in Action**
```java
public class DataTypes {
    public static void main(String[] args) {
        byte age = 30;                     // Small whole number
        long viewsCount = 3_000_000_000L;  // Large whole number
        float price = 10.99F;              // Decimal number
        char grade = 'A';                  // Single character
        boolean isEligible = true;         // Boolean value
    }
}
```

#### **Example 3: Type Conversion (Casting)**
```java
public class Casting {
    public static void main(String[] args) {
        int x = 10;
        double y = x; // Implicit casting (int → double)
        System.out.println(y); // Output: 10.0

        double a = 10.5;
        int b = (int) a; // Explicit casting (double → int)
        System.out.println(b); // Output: 10 (decimal part lost)
    }
}
```

1. **Variables** store data in memory. Declare with `type name = value;`.
2. **Primitive types** (`int`, `double`, `boolean`, etc.) store simple values.
3. **Reference types** (e.g., `String`) store complex objects.
4. Use **camelCase** for variable names (e.g., `studentName`).
5. **Initialize variables** before use.
6. **Clean code**: One variable per line, meaningful names, avoid abbreviations.

# **Java: Primitive vs. Reference Types & Strings**

## **Key Concepts**
- **Primitive Types**: Simple data types (e.g., `int`, `byte`, `boolean`) that store values directly.
- **Reference Types**: Complex objects (e.g., `Date`, `String`, `Point`) that store references (memory addresses) to objects.
- **Memory Management**:
  - Primitives store values in memory.
  - References store addresses of objects in memory.
- **String Class**: A reference type with special syntax for initialization (no `new` required).
- **Dot Operator (`.`)**: Used to access members (methods/fields) of objects.
- **Packages**: Organize classes into namespaces (e.g., `java.util`, `java.awt`).

---

## **Important Points**

### **1. Primitive vs. Reference Types**
| Feature          | Primitive Types (`int`, `byte`, etc.) | Reference Types (`Date`, `String`, etc.) |
|------------------|--------------------------------------|------------------------------------------|
| **Memory**       | Stores the actual value.             | Stores a reference (address) to an object. |
| **Initialization** | Direct assignment (`int x = 5;`).   | Requires `new` operator (e.g., `Date now = new Date();`). |
| **Members**      | No methods/fields.                   | Has methods/fields (e.g., `string.length()`). |
| **Copy Behavior** | Copies the value (independent).      | Copies the reference (both variables point to the same object). |

### **2. Memory Allocation**
- **Primitives**: Memory is automatically allocated/released by Java.
- **References**: Use `new` to allocate memory (Java handles deallocation via garbage collection).

### **3. Strings in Java**
- **Special Reference Type**: Can be initialized without `new` (e.g., `String s = "Hello";`).
- **Common Methods**:
  - `length()`: Returns the number of characters.
  - `endsWith(String suffix)`: Checks if the string ends with a given suffix.
  - `startsWith(String prefix)`: Checks if the string starts with a given prefix.
  - `concat(String str)`: Joins strings (or use `+` operator).

### **4. Packages**
- **Purpose**: Avoid naming conflicts (e.g., `java.util.Date` vs. `java.sql.Date`).
- **Importing**: Use `import` to use classes from other packages (e.g., `import java.util.Date;`).
- **`java.lang`**: Automatically imported (contains `String`, `System`, etc.).

---

## **Examples**

### **1. Primitive vs. Reference Initialization**
```java
// Primitive (no 'new' needed)
int age = 30;

// Reference (requires 'new')
Date now = new Date();
```

### **2. Copy Behavior**
```java
// Primitives: Independent copies
int x = 1;
int y = x;  // y = 1 (independent of x)
x = 2;      // y remains 1

// References: Same object
Point p1 = new Point(1, 1);
Point p2 = p1;  // p2 references the same object as p1
p1.x = 2;       // p2.x is now 2 (same object)
```

### **3. String Operations**
```java
String message = "Hello, World!";

// Concatenation
String excited = message + "!!";  // "Hello, World!!!"

// Methods
boolean endsWithExclamation = message.endsWith("!");  // true
int length = message.length();  // 13
```

---

## **Revision Summary**
1. **Primitive Types**: Store values directly (e.g., `int`, `boolean`).
2. **Reference Types**: Store references to objects (e.g., `Date`, `String`).
3. **Memory**:
   - Primitives: Values stored in memory.
   - References: Addresses stored; objects allocated with `new`.
4. **Strings**: Special reference type with shorthand initialization (`String s = "text";`).
5. **Packages**: Organize classes (e.g., `java.util.Date`).
6. **Dot Operator**: Accesses object members (e.g., `string.length()`).

**Key Takeaway**: Primitives are independent; references share objects. Strings are reference types but behave like primitives in initialization.

# **Java String and Array Methods: Study Notes**

---

## **Title**
**Java String Manipulation & Arrays: Key Methods and Concepts**

---

## **Key Concepts**
- **String Methods**: `length()`, `indexOf()`, `replace()`, `toLowerCase()`, `toUpperCase()`, `trim()`
- **String Immutability**: Strings in Java cannot be modified; methods return new strings.
- **Escape Sequences**: `\”`, `\\`, `\n`, `\t`
- **Arrays**: Declaration, initialization, accessing elements, `length` property, sorting, multi-dimensional arrays
- **Method Overloading**: Multiple methods with the same name but different parameters
- **Exceptions**: Errors reported during runtime (e.g., `ArrayIndexOutOfBoundsException`)

---

## **Important Points**

### **1. String Methods**
#### **a. `length()`**
- Returns the number of characters in a string.
- Useful for validation (e.g., checking username length).

#### **b. `indexOf()`**
- Returns the **index of the first occurrence** of a character or substring.
- Returns `-1` if the character/substring is not found.
- Example: `"Hello".indexOf('H')` returns `0`.

#### **c. `replace()`**
- Replaces a **target** character/substring with a **replacement**.
- **Parameters vs. Arguments**:
  - **Parameters**: Placeholders in method definition (e.g., `target`, `replacement`).
  - **Arguments**: Actual values passed (e.g., `"!"`, `"*"`).
- **Does not modify the original string** (returns a new string).

#### **d. `toLowerCase()` / `toUpperCase()`**
- Converts all characters to lowercase or uppercase.
- Returns a **new string** (original remains unchanged).

#### **e. `trim()`**
- Removes **leading and trailing whitespace**.
- Useful for cleaning user input (e.g., form fields).

---

### **2. String Immutability**
- Strings in Java **cannot be changed** after creation.
- Any method that "modifies" a string (e.g., `replace()`, `toLowerCase()`) **returns a new string**.
- Example:
  ```java
  String message = "Hello!";
  String newMessage = message.replace("!", "*");
  System.out.println(message); // Output: "Hello!" (unchanged)
  ```

---

### **3. Escape Sequences**
Used to include special characters in strings:
| Escape Sequence | Description                     | Example                     |
|-----------------|---------------------------------|-----------------------------|
| `\"`            | Double quote                    | `"Hello \"Mars\""` → `Hello "Mars"` |
| `\\`            | Backslash                       | `"C:\\Windows"` → `C:\Windows` |
| `\n`            | New line                        | `"Line1\nLine2"` → Line1<br>Line2 |
| `\t`            | Tab                             | `"Hello\tWorld"` → `Hello   World` |

---

### **4. Arrays**
#### **a. Declaration & Initialization**
- **Old Syntax**:
  ```java
  int[] numbers = new int[5]; // Array of size 5 (default values: 0)
  numbers[0] = 1; // Assign value to first element
  ```
- **New Syntax (if values are known)**:
  ```java
  int[] numbers = {2, 3, 5, 1, 4}; // Array of size 5
  ```

#### **b. Accessing Elements**
- Index starts at **0**.
- Example:
  ```java
  System.out.println(numbers[0]); // Output: 2
  ```
- **Invalid index** (e.g., `numbers[10]`) throws `ArrayIndexOutOfBoundsException`.

#### **c. `length` Property**
- Returns the **number of elements** in the array.
- Example:
  ```java
  System.out.println(numbers.length); // Output: 5
  ```

#### **d. Sorting Arrays**
- Use `Arrays.sort()` from `java.util.Arrays`.
- Example:
  ```java
  import java.util.Arrays;
  Arrays.sort(numbers); // Sorts the array in ascending order
  ```

#### **e. Multi-Dimensional Arrays**
- Used for matrices or cubes.
- Example (2D array):
  ```java
  int[][] matrix = new int[2][3]; // 2 rows, 3 columns
  matrix[0][0] = 1; // Assign value to first row, first column
  ```

---

## **Examples**

### **1. String Methods**
```java
String username = "Skywalker2024";

// Check length
if (username.length() > 20) {
    System.out.println("Username too long!");
}

// Find index of 'w'
int index = username.indexOf('w'); // Returns 3

// Replace characters
String newUsername = username.replace("2024", "2025"); // "Skywalker2025"

// Trim whitespace
String input = "  hello  ";
String trimmedInput = input.trim(); // "hello"
```

### **2. Escape Sequences**
```java
// Double quotes
String quote = "He said, \"Hello!\"";

// Backslash
String path = "C:\\Program Files\\Java";

// New line
String poem = "Roses are red,\nViolets are blue";

// Tab
String data = "Name:\tJohn";
```

### **3. Arrays**
```java
// Initialize array
int[] numbers = {5, 2, 8, 1, 3};

// Access element
System.out.println(numbers[2]); // Output: 8

// Sort array
Arrays.sort(numbers); // {1, 2, 3, 5, 8}

// Print array
System.out.println(Arrays.toString(numbers)); // "[1, 2, 3, 5, 8]"
```

---

## **Short Revision Summary**
1. **String Methods**:
   - `length()` → Check string length.
   - `indexOf()` → Find character/substring index.
   - `replace()` → Replace characters (returns new string).
   - `toLowerCase()`/`toUpperCase()` → Change case.
   - `trim()` → Remove whitespace.

2. **String Immutability**:
   - Strings **cannot be modified**; methods return new strings.

3. **Escape Sequences**:
   - `\"`, `\\`, `\n`, `\t` → Include special characters.

4. **Arrays**:
   - Fixed size; use `new` or `{}` for initialization.
   - Access elements via index (starts at 0).
   - `length` → Number of elements.
   - `Arrays.sort()` → Sort array.
   - Multi-dimensional arrays → Store matrices/cubes.

5. **Exceptions**:
   - Invalid array index → `ArrayIndexOutOfBoundsException`.

## **Key Concepts**

- Syntax: `(targetType) value`
- Example:


# **Structured Learning Notes: Conditional Statements in Java**

---

## **Title: Mastering Conditional Statements in Java**
**Topics Covered:**
- If, else-if, and else statements
- Simplifying boolean expressions
- Ternary operator
- Switch statements

---

## **Key Concepts**

- **If-Else Statements:**
  - Used to execute code based on boolean conditions.
  - Can have multiple `else-if` clauses and one optional `else` clause.
  - Braces `{}` are optional for single statements but recommended for clarity.

- **Boolean Simplification:**
  - Redundant conditions can be removed to simplify code.
  - Proper formatting improves readability.

- **Ternary Operator (`?:`):**
  - A concise way to write simple if-else conditions.
  - Syntax: `variable = (condition) ? valueIfTrue : valueIfFalse;`

- **Switch Statements:**
  - Used to execute different code blocks based on the value of a variable.
  - Requires `break` statements to prevent fall-through.
  - Supports `default` case for unmatched values.

---

## **Important Points**

### **1. If-Else Statements**
- **Structure:**
  ```java
  if (condition1) {
      // Code if condition1 is true
  } else if (condition2) {
      // Code if condition2 is true
  } else {
      // Code if no conditions are true
  }
  ```
- **Formatting Best Practices:**
  - Avoid unnecessary braces for single statements (controversial but common).
  - Align `else-if` and `else` clauses for symmetry:
    ```java
    if (condition) {
        // Code
    } else if (condition) {
        // Code
    } else {
        // Code
    }
    ```
- **Simplifying Conditions:**
  - If `condition1` is false, subsequent conditions can be simplified.
  - Example:
    ```java
    if (temperature > 30) {
        System.out.println("Hot day");
    } else if (temperature <= 30 && temperature > 20) {  // Redundant check
        System.out.println("Beautiful day");
    }
    ```
    Simplified to:
    ```java
    if (temperature > 30) {
        System.out.println("Hot day");
    } else if (temperature > 20) {  // No need to check <= 30
        System.out.println("Beautiful day");
    }
    ```

---

### **2. Variable Scope in If-Else Blocks**
- Variables declared inside a block `{}` are **scoped** to that block.
- Example of **incorrect** scoping:
  ```java
  if (income > 100_000) {
      boolean hasHighIncome = true;  // Only accessible here
  }
  System.out.println(hasHighIncome);  // Error: Variable not accessible
  ```
- **Solution:** Declare the variable outside the block:
  ```java
  boolean hasHighIncome = false;
  if (income > 100_000) {
      hasHighIncome = true;
  }
  ```

---

### **3. Simplifying Boolean Assignments**
- **Amateur Approach:**
  ```java
  boolean hasHighIncome;
  if (income > 100_000) {
      hasHighIncome = true;
  } else {
      hasHighIncome = false;
  }
  ```
- **Professional Approach:**
  ```java
  boolean hasHighIncome = (income > 100_000);  // Direct assignment
  ```
  - Parentheses are optional but improve readability.

---

### **4. Ternary Operator**
- **Syntax:**
  ```java
  variable = (condition) ? valueIfTrue : valueIfFalse;
  ```
- **Example:**
  ```java
  String className = (income > 100_000) ? "First" : "Economy";
  ```
- **Use Case:**
  - Best for simple if-else conditions where a variable is assigned based on a condition.

---

### **5. Switch Statements**
- **Structure:**
  ```java
  switch (variable) {
      case value1:
          // Code
          break;
      case value2:
          // Code
          break;
      default:
          // Code if no cases match
  }
  ```
- **Key Points:**
  - `break` is required to exit the switch block (otherwise, fall-through occurs).
  - `default` is optional but recommended for handling unexpected values.
  - Works with `int`, `short`, `byte`, `char`, `String`, and `enum` types.

- **Example:**
  ```java
  String role = "admin";
  switch (role) {
      case "admin":
          System.out.println("You are an admin");
          break;
      case "moderator":
          System.out.println("You are a moderator");
          break;
      default:
          System.out.println("You are a guest");
  }
  ```

---

## **Examples**

### **Example 1: If-Else Statement**

## **Short Revision Summary**

1. **If-Else Statements:**
   - Use `if`, `else-if`, and `else` to execute code based on conditions.
   - Simplify conditions by removing redundant checks.
   - Format code for readability (align clauses).

2. **Boolean Simplification:**
   - Directly assign boolean expressions to variables (e.g., `boolean x = (a > b);`).

3. **Ternary Operator:**
   - Use `?:` for concise if-else assignments.
   - Syntax: `variable = (condition) ? valueIfTrue : valueIfFalse;`.

4. **Switch Statements:**
   - Use `switch` to execute code based on variable values.
   - Always include `break` to avoid fall-through.
   - Use `default` for unmatched cases.

**Final Tip:** Prefer ternary operators for simple conditions and switch statements for multiple value checks. Always prioritize readability!

# **Study Notes: Control Flow in Java – FizzBuzz & Loops**

---

## **Title:**
**Control Flow in Java: Conditional Statements, FizzBuzz Problem, and Loops (for & while)**

---

## **Key Concepts**

- **Conditional Statements (`if`, `else if`, `else`)** – Execute different code blocks based on boolean conditions.
- **Modulus Operator (`%`)** – Returns the remainder of a division; used to check divisibility.
- **DRY Principle (Don’t Repeat Yourself)** – Avoid code duplication for maintainability.
- **Order of Conditions** – More specific conditions should be checked first to avoid logical errors.
- **Loops** – Repeat code execution:
  - **`for` loop** – Best when the number of iterations is known.
  - **`while` loop** – Best when iterations depend on a dynamic condition.
- **Nested vs. Flat Structure** – Trade-offs between readability and code duplication.

---

## **Important Points**

### **1. The FizzBuzz Problem**
A classic programming exercise often used in interviews:
- **Input:** A number.
- **Output:**
  - "Fizz" if divisible by **5**.
  - "Buzz" if divisible by **3**.
  - "FizzBuzz" if divisible by **both 5 and 3**.
  - The number itself if none of the above.

#### **Common Mistake: Incorrect Condition Order**
- If the condition for "FizzBuzz" (divisible by both 3 and 5) is not checked **first**, the program will incorrectly print "Fizz" or "Buzz" instead.
- **Solution:** Place the most specific condition (`number % 3 == 0 && number % 5 == 0`) at the top.

#### **Two Approaches to Solve FizzBuzz**
| **Approach 1: Flat Structure (Preferred)** | **Approach 2: Nested Conditions (Less Readable)** |
|--------------------------------------------|---------------------------------------------------|
| Checks conditions in order of specificity. | Uses nested `if-else` to avoid repeating conditions. |
| Cleaner and easier to read.                | Reduces code duplication but increases complexity. |
| Slight repetition of conditions.           | Harder to maintain with deeper nesting.           |

---

### **2. Loops in Java**
#### **a) `for` Loop**
- **Syntax:**
  ```java
  for (initialization; condition; update) {
      // Code to repeat
  }
  ```
- **Use Case:** When the number of iterations is known (e.g., print "Hello World" 5 times).
- **Example:**
  ```java
  for (int i = 0; i < 5; i++) {
      System.out.println("Hello World " + i);
  }
  ```
  - **Output:**
    ```
    Hello World 0
    Hello World 1
    Hello World 2
    Hello World 3
    Hello World 4
    ```

#### **b) `while` Loop**
- **Syntax:**
  ```java
  while (condition) {
      // Code to repeat
  }
  ```
- **Use Case:** When iterations depend on a dynamic condition (e.g., user input).
- **Example:**
  ```java
  Scanner scanner = new Scanner(System.in);
  String input = "";
  while (!input.equals("quit")) {
      System.out.print("Enter a command: ");
      input = scanner.nextLine();
  }
  ```
  - **Behavior:** Keeps asking for input until the user types "quit".

#### **Key Differences**
| Feature          | `for` Loop                          | `while` Loop                        |
|------------------|-------------------------------------|-------------------------------------|
| **Initialization** | Inside the loop declaration.       | Outside the loop.                   |
| **Update**       | Inside the loop declaration.       | Inside the loop body.               |
| **Best For**     | Known iterations.                  | Dynamic conditions.                 |

---

## **Examples**

### **1. FizzBuzz Solution (Flat Structure)**
```java
import java.util.Scanner;

public class FizzBuzz {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        System.out.print("Number: ");
        int number = scanner.nextInt();

        if (number % 3 == 0 && number % 5 == 0) {
            System.out.println("FizzBuzz");
        } else if (number % 5 == 0) {
            System.out.println("Fizz");
        } else if (number % 3 == 0) {
            System.out.println("Buzz");
        } else {
            System.out.println(number);
        }
    }
}
```

### **2. Printing Numbers in Reverse Using `for` Loop**
```java
for (int i = 5; i > 0; i--) {
    System.out.println("Hello World " + i);
}
```
- **Output:**
  ```
  Hello World 5
  Hello World 4
  Hello World 3
  Hello World 2
  Hello World 1
  ```

### **3. User Input Loop Using `while`**
```java
Scanner scanner = new Scanner(System.in);
String input = "";
while (!input.equals("quit")) {
    System.out.print("Enter a command (or 'quit' to exit): ");
    input = scanner.nextLine();
    System.out.println("You entered: " + input);
}
```
- **Behavior:** Repeats until the user types "quit".

---

## **Short Revision Summary**

1. **FizzBuzz Problem:**
   - Check divisibility by **3**, **5**, or **both** in the correct order.
   - **Flat structure (preferred)** is cleaner than nested conditions.

2. **Loops:**
   - **`for` loop:** Use when iterations are known (e.g., `for (int i = 0; i < 5; i++)`).
   - **`while` loop:** Use for dynamic conditions (e.g., `while (!input.equals("quit"))`).

3. **Key Takeaways:**
   - **Order matters** in `if-else` conditions (specific first).
   - **DRY Principle** avoids repetition but can lead to nested code.
   - **Trade-offs** exist in programming—choose readability over minor optimizations.

---

- Experiment with nested loops and loop control statements (`break`, `continue`).

# **Java Loops and Input Handling: Study Notes**

---

## **Title**
**Java Loops, Input Validation, and Control Flow**

---

## **Key Concepts**
- **String Comparison**: Use `.equals()` instead of `==` to compare string values.
- **Loop Types**:
  - `while` loop: checks condition before execution.
  - `do-while` loop: executes at least once, checks condition after.
  - `for` loop: used for known iteration counts.
  - `for-each` loop: simplified iteration over arrays/collections.
- **Loop Control Statements**:
  - `break`: exits the loop immediately.
  - `continue`: skips to the next iteration.
- **Infinite Loops**: Use `while(true)` with a `break` condition.
- **Input Validation**: Use loops to repeatedly prompt the user until valid input is received.
- **Scanner Best Practices**: Declare `Scanner` once outside loops to avoid memory issues.

---

## **Important Points**

### **1. String Comparison and Input Handling**
- **`.equals()` Method**:
  - Use `input.equals("quit")` instead of `input == "quit"` to compare string values.
  - The `==` operator compares memory addresses, not content.
- **Case Insensitivity**:
  - Convert input to lowercase using `.toLowerCase()` to handle mixed-case inputs (e.g., "Quit", "QUIT").
  - Example: `input = scanner.next().toLowerCase();`
- **Scanner Object**:
  - Declare the `Scanner` object **once** outside the loop to avoid memory pollution.
  - Example:
    ```java
    Scanner scanner = new Scanner(System.in);
    while (!input.equals("quit")) {
        input = scanner.next().toLowerCase();
    }
    ```

---

### **2. Loop Types and Use Cases**

#### **While Loop**
- **Syntax**:
  ```java
  while (condition) {
      // code to execute
  }
  ```
- **Behavior**:
  - Checks the condition **before** executing the loop body.
  - If the condition is `false` initially, the loop never runs.
- **Use Case**:
  - Ideal when the number of iterations is **unknown** (e.g., user input loops).

#### **Do-While Loop**
- **Syntax**:
  ```java
  do {
      // code to execute
  } while (condition);
  ```
- **Behavior**:
  - Executes the loop body **at least once** before checking the condition.
  - Useful for menus or prompts where the user must enter input at least once.
- **Use Case**:
  - Rarely used, but helpful when the loop must run **at least once** (e.g., game loops, input validation).

#### **For Loop**
- **Syntax**:
  ```java
  for (initialization; condition; update) {
      // code to execute
  }
  ```
- **Use Case**:
  - Best for **known iteration counts** (e.g., iterating over arrays, counting loops).
  - Example:
    ```java
    for (int i = 0; i < fruits.length; i++) {
        System.out.println(fruits[i]);
    }
    ```

#### **For-Each Loop**
- **Syntax**:
  ```java
  for (dataType item : array/collection) {
      // code to execute
  }
  ```
- **Behavior**:
  - Simplifies iteration over arrays/collections.
  - **Limitations**:
    - Forward-only (cannot iterate backward).
    - No access to the index of the current item.
- **Use Case**:
  - Ideal for reading all elements in an array/collection without needing the index.
  - Example:
    ```java
    for (String fruit : fruits) {
        System.out.println(fruit);
    }
    ```

---

### **3. Loop Control Statements**

#### **Break Statement**
- **Purpose**: Exits the loop immediately.
- **Use Case**:
  - Terminate a loop when a condition is met (e.g., user enters "quit").
  - Example:
    ```java
    while (true) {
        String input = scanner.next();
        if (input.equals("quit")) {
            break; // Exit the loop
        }
        System.out.println(input);
    }
    ```

#### **Continue Statement**
- **Purpose**: Skips the current iteration and moves to the next one.
- **Use Case**:
  - Skip specific values (e.g., ignore "pass" in user input).
  - Example:
    ```java
    while (!input.equals("quit")) {
        input = scanner.next();
        if (input.equals("pass")) {
            continue; // Skip printing "pass"
        }
        System.out.println(input);
    }
    ```

---

### **4. Infinite Loops**
- **Syntax**: `while (true)`
- **Use Case**:
  - Common in input validation where the loop runs until a valid input is received.
  - **Must include a `break` statement** to avoid infinite execution.
- **Example**:
  ```java
  while (true) {
      System.out.print("Enter a number (1000-1000000): ");
      int principal = scanner.nextInt();
      if (principal >= 1000 && principal <= 1000000) {
          break; // Valid input, exit loop
      }
      System.out.println("Invalid input. Try again.");
  }
  ```

---

### **5. Input Validation**
- **Purpose**: Ensure user input meets specific criteria (e.g., range, type).
- **Approach**:
  - Use an **infinite loop** (`while(true)`) to repeatedly prompt the user.
  - Validate input inside the loop and `break` if valid.
- **Example**:
  ```java
  while (true) {
      System.out.print("Enter principal ($1K-$1M): ");
      int principal = scanner.nextInt();
      if (principal >= 1000 && principal <= 1000000) {
          break;
      }
      System.out.println("Enter a value between 1,000 and 1,000,000.");
  }
  ```

---

## **Important Points**

### **1. Variable Scope and Compilation Errors**
- Variables declared inside a block (e.g., within `{ }`) are **scoped to that block** and **not accessible outside** of it.
- Example: If `principal` is declared inside a `while` loop, it cannot be used outside the loop, causing a **"cannot resolve symbol"** compilation error.
- **Solution**: Declare variables **outside the loop** (e.g., at the beginning of the method) to ensure they are accessible throughout the method.

### **2. Data Validation Using Infinite Loops**
- Use an **infinite loop** (`while (true)`) to repeatedly prompt the user until valid input is provided.
- Validate input using **conditional statements** (`if-else`).
- If input is valid:
  - Perform calculations (e.g., convert annual interest to monthly interest).
  - **Break out of the loop** using `break`.
- If input is invalid:
  - Print an **error message** and **repeat the loop**.

### **3. Example of Data Validation for Mortgage Calculator**
- **Principal Amount**: Must be between 1,000 and 1,000,000.
- **Annual Interest Rate**: Must be between 1% and 30%.
- **Loan Period (Years)**: Must be between 1 and 30 years.

### **4. Refactoring for Clean Code**
- Long methods with many statements are **hard to read and maintain**.
- **Solution**: Break down code into **smaller, reusable chunks** (e.g., separate methods for input validation, calculations, and output).
- Follow **Martin Fowler’s principle**:
  > *"Any fool can write code that a computer can understand. Good programmers write code that humans can understand."*

### **5. Control Flow Statements**
| **Type**               | **Purpose**                                                                 | **Examples**                          |
|------------------------|-----------------------------------------------------------------------------|---------------------------------------|
| **Conditional**        | Make decisions in code based on conditions.                                | `if`, `else`, `switch`                |
| **Loops**              | Execute code repeatedly.                                                   | `for`, `while`, `do-while`, `for-each`|
| **Jump Statements**    | Control loop execution (exit or skip iterations).                          | `break`, `continue`                   |

### **6. Comparison and Logical Operators**
- **Comparison Operators**: `==`, `!=`, `>`, `<`, `>=`, `<=`
- **Logical Operators**: `&&` (AND), `||` (OR), `!` (NOT)
- Used to **combine conditions** (e.g., `if (interest >= 1 && interest <= 30)`).
